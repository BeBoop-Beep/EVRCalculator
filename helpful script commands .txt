----------------------------
desirability score runner
----------------------------
1. pokeapi (get all pokemon)
python backend/scripts/ingest_pokemon_desirability.py --source pokeapi --commit

2. all favorite pokemon sourced
python backend/scripts/ingest_pokemon_desirability.py --source favoritepokemon --commit

3.pokemon google trends relative search interest:
python backend/scripts/ingest_pokemon_trends.py \
  --provider pytrends \
  --timeframe today 1-m \
  --commit \
  --delay-seconds 180 \
  --max-retries 1 \
  --retry-backoff-seconds 600 \
  --stop-after-consecutive-429s 1 \
  --cooldown-after-429-seconds 1800

4. Verify the latest today 1-m Google Trends snapshot has full coverage
select
  s.id as snapshot_id,
  s.timeframe,
  s.window_role,
  s.status,
  count(r.id) as source_row_count
from pokemon_trend_source_snapshots s
left join pokemon_trend_source_rows r
  on r.snapshot_id = s.id
where s.timeframe = 'today 1-m'
group by
  s.id,
  s.timeframe,
  s.window_role,
  s.status,
  s.captured_at
order by s.captured_at desc
limit 5;

Expected target:

source_row_count = 1025
status = captured_relative_search_interest

5. Derive Google Trends recent_trend_score from the latest complete today 1-m snapshot
python backend/scripts/ingest_pokemon_trends.py \
  --derive-from-existing \
  --provider pytrends \
  --commit

6. Verify recent_trend_score rows exist
select
  score_name,
  primary_snapshot_id,
  count(*) as score_count
from pokemon_trend_scores
where score_name = 'recent_trend_score'
group by score_name, primary_snapshot_id
order by primary_snapshot_id desc;

Expected target:

recent_trend_score | latest today 1-m snapshot_id | 1025

7. Build Pokémon Desirability Composite Score
python backend/scripts/build_pokemon_desirability_composite.py --commit --log-level INFO

8. Verify composite score rows
select
  scoring_version,
  count(*) as composite_count
from pokemon_desirability_composite_scores
group by scoring_version
order by scoring_version;

Expected target:

pokemon_desirability_composite_v1 | 1025

9. Inspect top composite results
select
  desirability_rank,
  pokedex_number,
  pokemon_name,
  fan_popularity_score,
  fan_popularity_rank,
  current_trend_score,
  current_trend_rank,
  desirability_score,
  desirability_tier
from pokemon_desirability_composite_scores
where scoring_version = 'pokemon_desirability_composite_v1'
order by desirability_rank
limit 25;


----------------------------
set-level desirability input rebuild (new set safe rerun)
----------------------------
Use this when cards are already in public.cards but canonical/desirability set rows are missing or stale.

single set dry-run (default):
python backend/scripts/build_pokemon_set_desirability_inputs.py --set chaosRising --log-level INFO

single set commit:
python backend/scripts/build_pokemon_set_desirability_inputs.py --set chaosRising --commit --log-level INFO

all sets dry-run:
python backend/scripts/build_pokemon_set_desirability_inputs.py --all --log-level INFO

all sets commit:
python backend/scripts/build_pokemon_set_desirability_inputs.py --all --commit --log-level INFO

This script chains:
1) pokemon_canonical_cards fallback upsert from public.cards (source='tcgplayer_cards_fallback')
2) pokemon_card_desirability_links builder
3) pokemon_set_hit_desirability_summaries builder
4) pokemon_set_desirability_component_scores builder
5) pokemon_set_opening_desirability_scores persistence (idempotent snapshot guard)

Pre/post coverage query (replace canonical key as needed):
select
  s.name as set_name,
  s.canonical_key,
  count(distinct c.id) as cards_rows,
  count(distinct pcc.id) as canonical_cards_rows,
  count(distinct pcdl.id) as desirability_link_rows,
  count(distinct pcdl.pokemon_reference_id) as linked_pokemon_count
from public.sets s
left join public.cards c on c.set_id = s.id
left join public.pokemon_canonical_cards pcc on pcc.set_id = s.id
left join public.pokemon_card_desirability_links pcdl on pcdl.pokemon_canonical_card_id = pcc.id
where s.canonical_key = 'chaosRising'
group by s.name, s.canonical_key;

Duplicate canonical guardrail:
select set_id, number, name, count(*) as dup_count
from public.pokemon_canonical_cards
where set_id = (select id from public.sets where canonical_key = 'chaosRising')
group by set_id, number, name
having count(*) > 1;

Duplicate API card id guardrail:
select pokemon_tcg_api_card_id, count(*) as dup_count
from public.pokemon_canonical_cards
where set_id = (select id from public.sets where canonical_key = 'chaosRising')
group by pokemon_tcg_api_card_id
having count(*) > 1;

Forbidden trainer/item/energy link guardrail (should return zero rows):
select
  pcc.name,
  count(pcdl.id) as link_count
from public.pokemon_canonical_cards pcc
left join public.pokemon_card_desirability_links pcdl
  on pcdl.pokemon_canonical_card_id = pcc.id
where pcc.set_id = (select id from public.sets where canonical_key = 'chaosRising')
  and pcc.name in (
    'Great Haul Net',
    'AZ''s Tranquility',
    'Energy Retrieval',
    'Special Red Card',
    'Bubbly W Energy',
    'Magnetic M Energy',
    'Nitro R Energy'
  )
group by pcc.name
having count(pcdl.id) > 0;

Required Pokemon link spot-check:
select
  pcc.name,
  count(pcdl.id) as link_count
from public.pokemon_canonical_cards pcc
left join public.pokemon_card_desirability_links pcdl
  on pcdl.pokemon_canonical_card_id = pcc.id
where pcc.set_id = (select id from public.sets where canonical_key = 'chaosRising')
  and pcc.name in (
    'Mega Greninja ex',
    'Mega Floette ex',
    'Mega Pyroar ex',
    'Mega Dragalge ex',
    'Cobalion ex',
    'Cinccino ex',
    'Gourgeist ex',
    'Keldeo',
    'Deoxys',
    'Meowstic',
    'Pumpkaboo'
  )
group by pcc.name
order by pcc.name;
